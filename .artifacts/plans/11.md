# Plan — Issue #11: palette-canon import scripts (Tailwind, Open Color, Catppuccin)

## Goal
Build 3 import scripts that fetch upstream sources and emit 6 palette atoms:
- `tools/imports/tailwind.ts`     → `palettes/tailwind-css/4.0.0/atom.yaml` (~242 swatches)
- `tools/imports/open-color.ts`   → `palettes/open-color/1.10.0/atom.yaml` (130 swatches)
- `tools/imports/catppuccin.ts`   → `palettes/catppuccin-latte/1.0.0/atom.yaml`
                                  + `catppuccin-frappe/1.0.0/atom.yaml`
                                  + `catppuccin-macchiato/1.0.0/atom.yaml`
                                  + `catppuccin-mocha/1.0.0/atom.yaml` (26 swatches each)

## Constraints (from schema + acceptance)
- YAML matches the `Palette` Zod schema (kind=palette, id, version, name, swatches[], modes.light.roles, modes.dark.roles, provenance).
- Idempotent: re-running produces no diff. Stable key ordering + deterministic swatch ordering + fixed `importedDate`.
- Real provenance: source URL, SPDX license, attribution.
- Light + dark role mappings on every atom. Sensibly invert single-mode flavors.
- `pnpm validate` passes.

## Architecture

```mermaid
flowchart TD
  T[tools/imports/tailwind.ts] -->|fetch JSON| TS[tailwindcss source colors]
  O[tools/imports/open-color.ts] -->|fetch JSON| OS[open-color.json]

  TS --> EY1[emit YAML]
  OS --> EY2[emit YAML]
  CS --> EY3[emit YAML x4]

  EY1 --> P1[palettes/tailwind-css/4.0.0/atom.yaml]
  EY2 --> P2[palettes/open-color/1.10.0/atom.yaml]
  EY3 --> P3[palettes/catppuccin-{latte,frappe,macchiato,mocha}/1.0.0/atom.yaml]
```

## Sub-tasks

### Seed (shared helper)
- Tiny `tools/imports/_shared.ts`: `fetchText`, `emitAtomYaml(filePath, data)` with deterministic key order matching the Palette schema (kind, id, version, name, description, tags, provenance, swatches, modes).

### T1 — Tailwind importer
- Source: pin a known stable upstream commit of tailwindcss's color file. Tailwind v4 uses `packages/tailwindcss/src/utilities/colors.ts` but the most parseable form is the legacy `src/util/colors.ts` (still kept in v3 maintenance branch and easier to parse). Use the actual current main: `https://raw.githubusercontent.com/tailwindlabs/tailwindcss/main/packages/tailwindcss/src/compat/colors.ts` — if fetch fails, fall back to the v3 path. Both files are TS objects mapping `name: { shade: hex }`. Parse by regex (no AST).
- Emit 22 families × 11 shades = 242 swatches (slate, gray, zinc, neutral, stone, red, orange, amber, yellow, lime, green, emerald, teal, cyan, sky, blue, indigo, violet, purple, fuchsia, pink, rose).
- Light roles: primary=blue-600, surface=white(slate-50), bg=white, text-primary=slate-900, success=green-600, error=red-600.
- Dark roles: primary=blue-400, bg=slate-900, surface=slate-800, text-primary=slate-100.
- Includes `black` and `white` as additional swatches.

### T2 — Open Color importer
- Source: `https://raw.githubusercontent.com/yeun/open-color/master/open-color.json`
- 13 hues × 10 shades = 130 swatches (gray, red, pink, grape, violet, indigo, blue, cyan, teal, green, lime, yellow, orange).
- Also exposes `white`, `black` as top-level — include those if present.
- IDs: `<hue>-<shade>` (e.g., `red-5`).
- Light: primary=blue-7, bg=white(gray-0), text=gray-9. Dark: primary=blue-5, bg=gray-9.

### T3 — Catppuccin importer
- Source: `https://raw.githubusercontent.com/catppuccin/palette/main/palette.json`
- Each of 4 flavors has 26 colors with hex (and rgb/hsl/accent flag). IDs: lowercase color names verbatim (`rosewater`, `flamingo`, `pink`, `mauve`, `red`, `maroon`, `peach`, `yellow`, `green`, `teal`, `sky`, `sapphire`, `blue`, `lavender`, `text`, `subtext1`, `subtext0`, `overlay2`, `overlay1`, `overlay0`, `surface2`, `surface1`, `surface0`, `base`, `mantle`, `crust`).
- Latte: light primary; dark mode inverts by using `mantle` as bg, `text` darker. Frappé/Macchiato/Mocha: dark primary; light mode uses base→crust hierarchy.
- Per Catppuccin's own light/dark guidance, each flavor still has the same accent names so role mappings can be uniform across flavors.

## Tests
- Each script's output validates against Palette schema.
- Tailwind atom has 242 swatches (or ±2 if upstream changes).
- Open Color atom has 130 swatches.
- Each Catppuccin atom has 26 swatches.
- Spot-check known hex values (e.g., Tailwind red-500 ≈ `#FB2C36` in v4 or `#EF4444` in v3; we record whatever upstream says today). Open Color blue-5 = `#339AF0`. Catppuccin Mocha mauve = `#CBA6F7`.
- Idempotency: re-run produces no file diff.
- Schema invariant: every role swatch-id resolves to a swatch.

## Output file ordering (deterministic)
- swatches in source order from upstream (preserve human-meaningful order).
- modes.light.roles and modes.dark.roles: keys alphabetically.
- Top-level YAML keys: kind, id, version, name, description, tags, provenance, swatches, modes (the standard atom order).

## Deps
None new. We use root's `yaml`, native `fetch` (Node 20+), `node:fs`.

## Risk
- Tailwind source path can move. Mitigation: try v4 path first, fall back to v3 maintenance path.
- The TS source file isn't strict JSON — needs careful regex parsing.
- Upstream version detection: hard-code `importedDate: 2026-05-17` + `importedFromVersion: <upstream-sha-or-tag>` for determinism. Fetch the actual commit SHA from the GitHub API and embed it.
